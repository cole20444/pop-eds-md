Deploy microservice OSGi configurations

AEM Guides needs two OSGi configurations to talk to the publishing microservice: one that wires up the IMS credential and one that points at the microservice endpoint. This task explains how to add them to your AEMaaCS project and deploy them through Cloud Manager.

PREREQUISITE
Before you begin:
  • You have completed Create Adobe Developer Console credentials and have the service account JSON available.
  • You have a working AEMaaCS project repository, with Cloud Manager pipelines that deploy on push to the appropriate branch.
  • You have Deployment Manager permissions in Cloud Manager so you can edit environment variables.
This task covers Tracks 2 and 3 of the three-track setup. Track 2 lives in Cloud Manager (the secret). Track 3 lives in your AEM project repository (the OSGi config files that reference the secret).
The split exists so secrets never have to be committed to source control. Cloud Manager holds the JSON; the OSGi config references it through the $[secret:NAME] placeholder syntax.
  1. In Cloud Manager, navigate to Environments<your environment>Configuration and add a new variable.
    1. Set the Name to SERVICE_ACCOUNT_DETAILS.
    2. Set the Type to Secret.
      ADDITIONAL INFORMATION: This ensures the value is encrypted at rest and is not visible after saving.
    3. Paste the entire service account JSON object (from the previous task) into the Value field.
    4. Click Save.
    STEP RESULT: The variable appears in the configuration list with its value redacted as ****.
  2. In your AEM project repository, navigate to the OSGi config folder for the target environment's runmode.
    ADDITIONAL INFORMATION: For an environment named dev, the path is ui.config/src/main/content/jcr_root/apps/<appId>/osgiconfig/config.author.dev/.
    For POP's AEM project the exact path is:
    ui.config/src/main/content/jcr_root/apps/aem-library/osgiconfig/config.author.dev/
  3. Create a file named com.adobe.aem.guides.eventing.ImsConfiguratorService.cfg.json with the following content:
    {

      "service.account.details": "$[secret:SERVICE_ACCOUNT_DETAILS]"

    }



    ADDITIONAL INFORMATION: This config wires the IMS configurator to read the JSON out of the Cloud Manager secret at runtime. Notice the OSGi $[secret:NAME] placeholder — that syntax is resolved by AEMaaCS at config load time.
  4. Create a file named com.adobe.fmdita.publishworkflow.PublishWorkflowConfigurationService.cfg.json with the following content:
    {

      "dxml.publish.microservice.url": "https://adobeioruntime.net/api/v1/web/543112-guidespublisher/default/publishercaller.json",

      "dxml.use.publish.microservice": true,

      "dxml.use.publish.microservice.native.pdf": true

    }







    ADDITIONAL INFORMATION: This config tells AEM Guides to use the publishing microservice (instead of in-process DITA-OT) and where to find it. The native.pdf flag enables native PDF output through the same microservice.
    ADDITIONAL INFORMATION:
    Note: The microservice URL is the same for every customer — it is Adobe's central worker, not something you host yourself.
  5. Commit and push both files to the branch that feeds your target Cloud Manager pipeline.
    ADDITIONAL INFORMATION: For a dev environment that usually means a develop branch; for stage and prod it's typically a release branch and main.
  6. In Cloud Manager, run the deploy pipeline for the target environment.
    ADDITIONAL INFORMATION: Watch the pipeline output for any OSGi configuration errors. If the secret was misspelled or missing, the ImsConfiguratorService will log a deserialization warning at startup.
    STEP RESULT: The build picks up the new OSGi configs, bundles them into the AEM package, and deploys them. A deploy on a small project takes 15–25 minutes.
  7. Verify the configuration by visiting /system/console/configMgr on the AEM author tier and searching for PublishWorkflowConfigurationService.
    ADDITIONAL INFORMATION: You should see the microservice URL populated and the Use Publish Microservice flag enabled.
Result
AEM Guides is now configured to publish via the Adobe microservice. The next time an author clicks Generate Output on a map with an EDS output preset, AEM will:
  1. Read SERVICE_ACCOUNT_DETAILS from the environment.
  2. Authenticate to Adobe IMS using the OAuth credential.
  3. Call the publishing microservice URL with the IMS access token.
  4. Wait for the microservice to commit DITA-OT HTML to the configured GitHub repository.

The expected configuration tree

After this task your AEM project repository should contain:
ui.config/src/main/content/jcr_root/apps/<appId>/osgiconfig/

└── config.author.dev/

    ├── com.adobe.aem.guides.eventing.ImsConfiguratorService.cfg.json

    └── com.adobe.fmdita.publishworkflow.PublishWorkflowConfigurationService.cfg.json





For stage and prod environments, copy the same two files into config.author.stage/ and config.author.prod/ respectively. The Cloud Manager secret needs to exist on each corresponding environment as well.
AFTER COMPLETING THE TASK
With the AEM side wired up, move on to the GitHub side: Create and configure the EDS repository.
