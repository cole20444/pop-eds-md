Configure a publish profile and output preset

A publish profile holds the credentials AEM Guides uses to commit to your GitHub repository. An output preset binds that profile to a specific ditamap. This task walks through creating both, with a workaround for the OAuth UI bug that affects Chromium-based browsers.

Before you begin:

  • You have completed Create and configure the EDS repository and have the repository name and owner handy.
  • You have completed Set up the DITA HTML transform Action.
  • You have administrator permissions on the AEM Guides author tier.
  • You can create a GitHub Personal Access Token (PAT) with repo scope, OR you have an OAuth App with Client ID and Client Secret.

The AEM Guides UI offers a Login button that is supposed to walk you through a GitHub OAuth device-code flow. In Chromium browsers — Chrome, Edge, Arc, Brave — the button hangs on Connection Failed due to Chrome's Cross-Origin-Opener-Policy (COOP) enforcement. See Known issues and troubleshooting for the technical detail.

The workaround is to create the publish profile via a direct API call instead of the UI. That is the path this task documents.

  1. Generate a GitHub Personal Access Token with repo scope.
    1. In GitHub, navigate to SettingsDeveloper settingsPersonal access tokensTokens (classic).
    2. Click Generate new token and pick the classic variant.
    3. Give the token a descriptive name (e.g., aem-guides-eds), set an expiration, and check the repo scope.
    4. Click Generate token and copy the value.
      Caution
      The token is shown only once. Save it directly to a password manager — do not paste it into chat, an editor scratch buffer, or any document, even temporarily.
  2. Capture an AEM login session cookie.The API call requires authentication. The easiest method is to extract the login-token cookie from a logged-in AEM browser tab.
    1. Sign in to your AEMaaCS author tier in a browser.
    2. Open DevTools → Application tab → Cookies → your AEM host.
    3. Copy the value of the login-token cookie.
  3. Run the publish-profile creation script.The script lives at /tmp/save-eds-publish-profile-aemaacs.sh in this project. It prompts for the AEMaaCS URL, login-token cookie, GitHub PAT, and OAuth App Client ID/Secret, then POSTs to /bin/guides/v1/thirdparty/profile.
    bash /tmp/save-eds-publish-profile-aemaacs.sh
    On success the API returns HTTP 200 with the new profile's ID. The profile now appears in AEM Guides under ToolsGuidesConfigurationPublish profiles.
  4. Open the ditamap you want to publish and switch to the Output Presets tab in the map console.You can reach Map Console from the asset browser by right-clicking a map and selecting Open in Map Console.
  5. Click the + icon to add a new preset.
  6. Configure the preset:
    Setting
    Value
    Type
    Select Edge Delivery Service (Beta).
    Name
    A descriptive name such as EDS – production.
  7. Click Add to create the preset, then open it and switch to the Config tab.
  8. Select your newly created publish profile from the Publish profile dropdown.
  9. Decide whether to enable Push to live.
    Tip
    For the very first publish, leave Push to live unchecked. That way you can verify the preview before promoting to the live URL. Subsequent publishes can flip it on once you are confident in the pipeline.
  10. Click Save.The map is now bound to the EDS preset and is ready to publish.

The publish profile and output preset are configured. Clicking Generate Output on the map will trigger the full pipeline described in Architecture and pipeline.

The publish profile API payload

For reference, the JSON payload the script posts looks like this (with secrets removed):

{
  "type": "github",
  "profileName": "EDS - pop-eds-md",
  "branch": "main",
  "repoName": "pop-eds-md",
  "rootFolder": "docs",
  "tokenCreationTime": 1716240000000,
  "userName": "cole20444",
  "workerURL": "https://dita-franklin-worker.adobeaem.workers.dev/cole20444/pop-eds-md/main/docs",
  "clientId": "",
  "clientSecret": "",
  "oauthToken": ""
}

Two field-name gotchas worth highlighting: the property is profileName (not name), and tokenCreationTime must be milliseconds since epoch.
