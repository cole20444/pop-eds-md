DITA element rendering reference

A comprehensive showcase of how individual DITA elements render through the EDS pipeline. Use this topic as a regression target whenever you change the transform script — every section below should still produce the expected output on the live site.

How this reference is organized

Each section below isolates one category of DITA elements, shows a representative example, and notes any transform-specific behavior. The first time an element appears in the example, it is wrapped in <b>tags</b> for clarity.

Inline emphasis

DITA's highlighting domain provides bold (b), italic (i), underline (u), monospace (tt), superscript like x2, and subscript like H2O. All six map to standard HTML inline elements through the transform.

For semantic emphasis, prefer term (for first-use of a domain word), keyword (for product or feature names), and cite (for titles of cited works). The transform preserves the class hint where helix-html2md does not strip it.

Inline technical content

For technical content, use:

  • uicontrol for buttons, menu items, and tab names. Multi-step menu paths use menucascade: FileOpen recentfstab.yaml.
  • filepath for paths on disk or inside a repository, like scripts/transform-dita.mjs.
  • codeph for inline code fragments such as npm install.
  • userinput for what the user types and systemoutput for what the system displays back.
  • cmdname for command names, varname for variable names, and parmname for parameter names.
  • MSG-1042 for message numbers and
    An accompanying message block can give

    
                    multi-line context about an error.

Paragraphs, line breaks, and quotes

A paragraph (<p>) is the simplest block element. Two adjacent paragraphs produce two visible paragraphs in the rendered output.

For a freestanding inline quote (e.g., a pull quote), use q for short quotations like this one. For longer quotations, use <lq>:

helix-html2md is fundamentally a translation exercise: HTML in, markdown out. Anything the converter cannot map cleanly to markdown is silently dropped, so the input HTML has to be very clean to begin with.

Long quotes get wrapped in <blockquote> by the transform and inherit the POP modernization layer's blockquote styling.

Unordered list (ul):

  • Another bullet, with

  1. First numbered item.
  2. Second numbered item, with
    1. a numbered sub-item,
    2. and another numbered sub-item.
  3. Third numbered item.

Notes and admonitions

The DITA <note> element has five type variants we support today. Each renders as a distinct POP-branded block.

Note
Plain note. Use for non-critical clarifications.
Warning
Warning. Use when ignoring the instruction could cause data loss or service downtime.
Tip
Tip. Use to share an optimization or shortcut that is not strictly required.
Caution
Caution. Use when the user might encounter unexpected behavior if they proceed without care.
Important
Important. Use to call out a step that is easy to miss but consequential.
Note
Note type=other — useful for custom variants. The transform Action falls back to a generic note class when it doesn't recognize the value.

Code blocks and preformatted text

For multi-line code, use <codeblock> with an outputclass that names the language:

interface PublishProfile {
  type: 'github';
  profileName: string;
  branch: string;
  repoName: string;
  rootFolder: string;
  tokenCreationTime: number;
  userName: string;
}
async function createProfile(p: PublishProfile): Promise {
  const res = await fetch('/bin/guides/v1/thirdparty/profile', {
    method: 'POST',
    headers: { 'content-type': 'application/json' },
    body: JSON.stringify(p),
  });
  if (!res.ok) throw new Error(`Profile create failed: ${res.status}`);
  return (await res.json()).profileId;
}

Examples

The <example> element wraps long-form examples with a title and supporting text. Note that <example> must be a direct child of <refbody> (or <body> in concepts and tasks) — it is not allowed inside a <section>. The example immediately following this section demonstrates the correct placement.

One-line preview refresh

To force aem.page to re-ingest a single path, POST to the admin API with an empty body:

curl -X POST \
  -H "Content-Length: 0" \
  "https://admin.hlx.page/preview///main/"

A 200 response means the content-bus has been invalidated for that path.

Logical grouping with sectiondiv

The <sectiondiv> element exists purely as a grouping container. It has no semantics of its own — useful when you need to attach a conditional-processing attribute (audience, props) to a chunk of content without forcing a new section.

Inside <refbody> or <body> the equivalent element is <bodydiv>; inside a <section> you must use <sectiondiv>. Multiple paragraphs inside a single <sectiondiv> are rendered with no visible wrapper. The transform Action unwraps them by default, so they end up as flat content in the resulting markdown.
