Set up the DITA HTML transform Action

DITA-OT's HTML output is not in the shape that helix-html2md can convert. This task adds a GitHub Action that runs on every push to docs/ and cleans the HTML into the magic <main><div>…</div></main> shape aem.live requires.

Before you begin:

  • You have completed Create and configure the EDS repository.
  • You have write access to the .github/workflows/ directory of the repository.
  • You have Node.js installed locally for testing the transform script before committing it.

The transform Action does two things. First, it runs a Node.js script (scripts/transform-dita.mjs) that uses cheerio to rewrite DITA-OT's HTML into clean EDS-compatible HTML. Second, it commits the result back to the same /docs/ directory.

The script is intentionally located outside/docs/ so the publishing microservice never overwrites it.

  1. Create the workflow file at .github/workflows/transform-dita.yml.
    name: Transform DITA HTML for EDS
on:
  push:
    branches: [main]
    paths:
      - 'docs/**.htm'
      - 'docs/**.html'
      - 'scripts/transform-dita.mjs'
      - '.github/workflows/transform-dita.yml'
permissions:
  contents: write
jobs:
  transform:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm install --no-save cheerio
      - run: node scripts/transform-dita.mjs
      - name: Commit cleaned HTML
        run: |
          git config user.name  "github-actions[bot]"
          git config user.email "41898282+github-actions[bot]@users.noreply.github.com"
          git add docs/
          if git diff --cached --quiet; then
            echo "No changes to commit"
          else
            git commit -m "Clean DITA HTML for EDS rendering"
            git push
          fi
    The paths filter ensures the Action only runs on relevant changes. Pushes by the Action itself (using GITHUB_TOKEN) do not retrigger workflows, so a skip-token guard is not required.
  2. Create the transform script at scripts/transform-dita.mjs.The script walks every .htm or .html file in /docs/, loads it with cheerio, applies a sequence of cleanup operations, and writes the result back as .html.

    The full script is in the POP repository at scripts/transform-dita.mjs. A condensed structure looks like:

    import { readFile, writeFile, readdir, rename, unlink } from 'node:fs/promises';
import { join, extname } from 'node:path';
import * as cheerio from 'cheerio';
const DOCS_DIR = 'docs';
const KEEP_CLASSES = new Set(['table']);
async function* walk(dir) {
  for (const entry of await readdir(dir, { withFileTypes: true })) {
    const full = join(dir, entry.name);
    if (entry.isDirectory()) yield* walk(full);
    else yield full;
  }
}
function transform(html) {
  const $ = cheerio.load(html, { decodeEntities: false });
  // 1. strip DITA-OT cruft
  $('.prolog, .breadcrumb, .prefix-content').remove();
  // 2. convert div-tables to EDS table blocks
  $('div.tablenoborder, div.table').each((_, el) => {
    // rewrite rows to nested divs, set class="table"
  });
  // 3. codeblock → 
    
  $('code.codeblock').each((_, el) => { /* ... */ });
  // 4. note variants → semantic block classes
  $('div.note').each((_, el) => {
    const variant = ($(el).attr('class') || '')
      .match(/note_(warning|tip|caution|important)/)?.[1];
    $(el).attr('class', variant || 'note');
  });
  // 5. inline span normalization
  $('span.uicontrol').each((_, el) => $(el).replaceWith(`${$(el).html()}`));
  $('span.filepath').each((_, el) => $(el).replaceWith(`${$(el).html()}`));
  // 6. unwrap structural divs/spans
  $('div, span').each((_, el) => {
    const cls = $(el).attr('class');
    if (!cls || !KEEP_CLASSES.has(cls)) {
      // unwrap if it has no semantic value
    }
  });
  // 7. wrap body in the helix-html2md magic shape
  const body = $('body').html();
  $('body').html(`
    ${body}
    `);
  return $.html();
}
for await (const file of walk(DOCS_DIR)) {
  if (!/\.html?$/.test(file)) continue;
  const cleaned = transform(await readFile(file, 'utf8'));
  const outPath = file.replace(/\.htm$/, '.html');
  await writeFile(outPath, cleaned);
  if (outPath !== file) await unlink(file);
}
    Tip
    Test the script locally against a representative DITA-OT HTML file before committing it. Run node scripts/transform-dita.mjs from the repository root and inspect the resulting file with a browser to confirm the output looks correct.
  3. Commit and push both files.
    git add .github/workflows/transform-dita.yml scripts/transform-dita.mjs
git commit -m "Add DITA HTML transform Action"
git push
  4. Trigger a test publish from AEM Guides and watch the Action run in the Actions tab of the GitHub repository.On a successful run you will see two commits land on main: first the Publishing for franklin commit from the microservice (with raw DITA-OT HTML), and seconds later a Clean DITA HTML for EDS rendering commit from the Action.

Every map publish now flows through the transform automatically. The HTML that aem.live ingests is in the <main><div>…</div></main> shape, DITA-OT cruft is gone, and tables, notes, and code blocks are normalized to clean EDS-friendly markup.

Before and after — note element example

Before the transform, DITA-OT emits a warning note like this:

Warning
Do not delete fstab.yaml.

After the transform, the same content is:

Do not delete fstab.yaml.

aem.live then renders that as the POP-branded Warning block defined in /blocks/warning/.
