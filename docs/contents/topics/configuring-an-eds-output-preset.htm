Configuring an EDS Output Preset

This task explains how to configure an Edge Delivery Service output preset on a DITA map in AEM Guides, so generated content is published to a GitHub repository served by aem.live.

Author: POP Tech Team

PREREQUISITE
Before you begin, ensure that:
  • You have a GitHub repository created from the aem-guides-boilerplate template.
  • AEM Code Sync is installed on that repository.
  • The microservice OSGi configurations are deployed to your AEMaaCS environment.
  • You have administrator permissions in AEM Guides.
An EDS output preset connects a DITA map to a publish profile, which holds the GitHub credentials that AEM Guides uses to commit generated HTML to your repository. Once configured, clicking Generate Output triggers the full publishing pipeline.
  1. Open your DITA map in Map Console.
    ADDITIONAL INFORMATION: You can access Map Console from the asset browser by right-clicking a map and selecting Open in Map Console.
  2. Navigate to the Output Presets tab.
  3. Click the + icon to create a new preset.
  4. In the preset creation dialog, configure the following options:
    • Type: select Edge Delivery Service (Beta).
    • Name: enter a descriptive name such as EDS – production.
  5. Click Add to create the preset.
    STEP RESULT: The new preset appears in your map's preset list.
  6. Open the newly created preset and switch to the Config tab.
    1. Select your publish profile from the Publish profile dropdown.
    2. Enable the Push to live toggle.
    3. Click Save.
  7. Click Generate Output to trigger the publishing pipeline.
    ADDITIONAL INFORMATION: The microservice converts your DITA content and commits it to the configured GitHub repository under /docs/.
Result
Your DITA content is now published to GitHub and available on both the EDS preview and live URLs:
  • https://main--{repo}--{owner}.aem.page/ (preview)
  • https://main--{repo}--{owner}.aem.live/ (live)
Tip: Use the preview URL to validate changes before they go live to end users.

Example publish profile JSON

For reference, here is what a complete publish profile looks like when stored in AEM:
{

  "profileId": "abc-123",

  "type": "github",

  "profileName": "EDS - production",

  "repoName": "my-eds-site",

  "userName": "my-org",

  "branch": "main",

  "rootFolder": "docs"

}
