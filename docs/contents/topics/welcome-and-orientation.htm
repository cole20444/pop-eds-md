Welcome and orientation

This guide walks you end-to-end through connecting AEM Guides to Edge Delivery Services (EDS), based on the working POP POC pipeline. Start here if you are new to the integration.
You are reading the master tutorial for the AEM Guides → EDS integration. It is written against the exact configuration that runs at POP today, so every step, file path, and command in this guide is real and verifiable.
By the end you will be able to: stand up the Adobe-side credentials, configure the AEM publishing microservice, prepare a GitHub repository, install the DITA HTML transform, and publish DITA content from AEM Guides to an aem.live preview URL.

Who this guide is for

This guide is written for:
  • AEM developers who maintain an AEM as a Cloud Service project and need to deploy the publishing microservice OSGi configurations.
  • DevOps engineers who manage Cloud Manager environments and GitHub repositories.
  • Content engineers who author DITA in AEM Guides and need to understand how their content reaches the public site.
  • Solution architects evaluating whether EDS is a good fit for a DITA-based documentation site.
You do not need deep knowledge of all three areas — the guide is organized so you can read just the sections relevant to your role.

What you will build

The end result is a content pipeline with these properties:
Authors edit DITA topics inside AEM Guides as they already do today.
One click on Generate Output publishes the whole map to a static site.
The static site is served from a global edge network (Cloudflare, via aem.live).
The frontend (CSS, JavaScript, blocks) lives in a GitHub repository that your team controls and reviews through normal pull requests.
For a complete architectural picture, jump ahead to Architecture and pipeline. If you would rather start clicking immediately, skip to Create Adobe Developer Console credentials.

What you need before starting

The detailed prerequisites are listed at the start of each task topic, but at a high level you need access to:
  • An Adobe organization with AEM as a Cloud Service.
  • A GitHub account and a repository owner you control.
  • A code editor and a terminal with git and node installed.
  • Roughly two hours of uninterrupted time for the first setup pass.
Tip: If you only have access to a sandbox AEMaaCS instance, you can still complete the entire pipeline — generated content will land in your GitHub repository and aem.page preview URLs just as it would on a production environment.
