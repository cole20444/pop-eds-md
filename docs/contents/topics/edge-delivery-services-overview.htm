What is EDS?

Adobe Edge Delivery Services is a fast, modern web delivery platform. It serves content from a GitHub repository through a globally distributed edge network. The system is built around three core ideas: GitHub-based authoring, markdown content, and block-based UI components.
For step-by-step setup instructions, see Configuring an EDS Output Preset.

Key features

  • Lightning-fast page loads via global edge caching.
  • Content stored as markdown in a Git repository.
  • Block-based component system for UI flexibility.
  • Seamless integration with AEM Guides for DITA content.

Supported output types

  1. HTML5 — the primary output for EDS sites.
  2. Native PDF — via Adobe's native PDF generator.
  3. JSON — for headless content consumption.
  4. CUSTOM — user-defined output via templates.

Terminology

Section
A logical grouping of content, separated by --- in markdown.
Block
A reusable UI component, encoded as a markdown table whose first row contains the block name.
fstab.yaml
The file that tells aem.live where to fetch site content from.

Important notes

Note: This is a regular note. Use it to draw attention to important but non-critical information.
Warning: Do not delete fstab.yaml from your repository — aem.live requires it for site registration.
Tip: Use the AEM Sidekick browser extension to publish individual pages without redeploying the whole site.
Caution: Changes to fstab.yaml after AEM Code Sync is installed may not be picked up.
Important: Always commit a valid mountpoint before installing the AEM Code Sync GitHub App on your repository.
