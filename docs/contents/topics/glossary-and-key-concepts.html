Glossary and key concepts

Definitions of the terms used throughout this guide. Where a term has a non-obvious origin (an Adobe codename, a Cloudflare worker name, a legacy product), the entry calls that out.

How to use this glossary

Terms are grouped by area (Adobe ecosystem, EDS / aem.live, DITA authoring, infrastructure). Within each group, terms are listed alphabetically. Cross-references to other terms use italics.

Adobe ecosystem

Term
Definition
Notes
AEM Guides
The Adobe Experience Manager DITA-based component content management system. Adds DITA authoring, map management, and structured-output publishing on top of AEM.
Formerly known as XML Documentation Add-on and before that fmDITA; you may see all three names in OSGi config keys.
AEMaaCS
AEM as a Cloud Service — Adobe's hosted, autoscaling version of AEM.
Required for the publishing microservice to run; the on-prem SDK cannot host FRANKLIN output.
Cloud Manager
The pipeline and environment management UI for AEMaaCS. Holds environment variables, deployment pipelines, and runtime configuration.
The SERVICE_ACCOUNT_DETAILS secret used by the microservice lives here.
Developer Console
Adobe's portal for creating OAuth credentials and registering integrations with Adobe APIs.
Found at developer.adobe.com/console.
IMS
Adobe's Identity Management System — the OAuth/OpenID Connect provider that issues access tokens to Adobe services.
The microservice exchanges the OAuth Server-to-Server credential for an IMS access token on every publish.
Publishing microservice
An Adobe-hosted service that runs DITA-OT inside an isolated container and writes the output directly to GitHub.
The endpoint URL is configured in the OSGi config PublishWorkflowConfigurationService.

EDS and aem.live

Term
Definition
Notes
Edge Delivery Services (EDS)
Adobe's edge-served, Git-backed delivery platform. Content is markdown in a Git repository; delivery is via a global CDN.
Sometimes called "Project Helix" or "Franklin" in older documentation.
aem.live
The runtime that turns the Git-backed content into rendered pages. aem.page serves preview URLs, aem.live serves published URLs.
Both URLs share the same source repository and content-bus; the difference is the preview/publish flag.
helix-html2md
Adobe's open-source HTML-to-markdown converter. aem.live runs it on any source whose fstab mountpoint declares type: markup.
Requires a specific HTML shape — see Known issues and troubleshooting.
content-bus
aem.live's internal caching layer. Stores the markdown representation of each page keyed by webPath.
Force a refresh with a POST to admin.hlx.page/preview/....
block
A reusable UI component encoded as a markdown table whose first cell names the block. Rendered by aem.live as <div class="block-name">.
Each block has its own /blocks//.css and /blocks//.js.
section
A logical grouping of content separated by --- in markdown. Renders as a <div class="section">.
Section metadata (style, name) lives in a special markdown table.
fstab.yaml
The file in the EDS repository root that tells aem.live where to fetch site content from.
Snapshot at AEM Code Sync install-time — see Create and configure the EDS repository.
AEM Code Sync
A GitHub App that registers a repository with aem.live and gives it preview/publish URLs.
Captures fstab.yaml at install-time; editing fstab afterwards does not update the captured value.
Sidekick
The browser extension authors use to preview and publish individual pages from an EDS site.
Not used in this pipeline because publishing is map-driven from AEM Guides.

DITA authoring

Term
Definition
Notes
DITA
Darwin Information Typing Architecture — an OASIS XML standard for technical documentation.
Topic-oriented, with strong typing (concept, task, reference, troubleshooting).
topic
A standalone unit of content, typed as concept, task, reference, or troubleshooting.
Each topic is its own .dita file.
ditamap
An XML file that defines the structure of a publication by referencing topics.
The unit of publishing — one Generate Output action operates on one map.
conref
Content reference — a mechanism for pulling content from one topic into another at build time.
Used in this guide for shared prerequisites; see shared-snippets.dita.
DITA-OT
The DITA Open Toolkit — the open-source engine that converts DITA into output formats.
Adobe's publishing microservice wraps DITA-OT; we never run it directly.

Infrastructure

Term
Definition
Notes
OAuth Server-to-Server
An Adobe IMS credential type designed for backend integrations. Replaces the deprecated JWT Service Account credential.
What AEM uses to authenticate to the publishing microservice.
GitHub Pages
GitHub's free static site hosting, served from a branch or a /docs/ directory.
The mountpoint source for this pipeline.
GitHub Action
GitHub's CI/CD primitive — a YAML workflow triggered by repository events.
The DITA HTML transform runs as a GitHub Action on every push to docs/.
cheerio
A Node.js library that implements a subset of jQuery's API for server-side HTML manipulation.
The DITA HTML transform script uses cheerio for selector-based traversal and rewriting.
